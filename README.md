# TMC空客组工程调查看板

纯前端工程调查可视化看板，用于实时监控TMC空客组工程调查项目进度。

## 功能特性

### 1. 统计卡片
顶部展示五组关键数字：总计、进行中、超期、注意、正常。

### 2. 三色分区
根据 `count`（计数）与 `investigation_days`（调查天数）的比值将项目分为三个区间：

| 区域 | 颜色 | 条件 | 含义 |
|------|------|------|------|
| 🔴 超期 | 红色 | count > 调查天数 | 计数已超过调查天数，需紧急处理 |
| 🟠 注意 | 橙色 | count 在 2/3~3/3 区间 | 计数接近调查天数，需关注 |
| 🟢 正常 | 绿色 | count ≤ 调查天数×2/3 | 计数在可控范围内 |

每个区间可点击展开/折叠，展示对应项目的卡片列表。卡片显示飞机注册号、机型、基地、ATA章节、调查状态、待办标签、故障分类标签、已过天数。点击卡片进入详情页。

### 3. ATA章节工程调查统计
- **总览柱状图**：按当前筛选条件统计各ATA章节的工程调查数量，点击柱条可查看该章节的故障原因饼图。
- **A350 柱状图**：固定筛选机型为A350的ATA章节统计（紫色柱）。
- **A330 柱状图**：固定筛选机型为A330的ATA章节统计（青色柱）。
- 点击饼图扇区可快速设置ATA章节和故障原因筛选条件，跳转至看板。

### 4. 工程师工程调查统计
堆叠柱状图展示每位TMC工程师的调查分布（超期/注意/正常/已完成），点击柱条可直接筛选该工程师。

### 5. 多维度筛选
支持按以下维度筛选：
- 故障类型（出勤故障/世界机队/SDR事件/重要故障/机组关注/重复故障/其他）
- 机型
- 飞机注册号（支持搜索）
- 执管基地
- ATA章节
- TMC工程师
- 故障原因
- 待办事项（有/无）
- 开始日期范围

筛选器支持多选，选中后标签变蓝。每个筛选项支持独立重置。

### 6. 待办事项看板
在筛选条件中将"待办事项"选为"有"后，右侧出现紫色"待办看板"按钮。点击弹出三列表格弹窗：

| 工程调查名称 | 待办事项 | 工程师 |
|-------------|---------|--------|
| fault_des | WG跟踪事项 | TMC工程师 |

表格中每条待办事项显示标签：
- `已存在待办事项xx天` — WG会议后经过的天数
- `待办事项由可靠性跟踪` — 已提交可靠性周会跟进
- `可靠性周会日期xxxx，展示项为WG跟踪事项` — 有可靠性周会日期
- `展示项为WG跟踪事项` — 无可靠性周会日期（纯文字）

表头固定不动，仅内容区域滚动。点击表格行可跳转至对应工程调查详情页，从详情页返回时自动回到待办看板。

### 7. 详情页
展示完整的调查信息：
- **流程进度**：调查中 → 科室讨论 → WG周会 → 可靠性周会
- **基本信息**：飞机注册号、机型、基地、故障日期、开始/结束日期、故障来源
- **故障信息**：ATA章节、故障描述
- **调查信息**：TMC/基地工程师、报告标题、评分
- **会议记录**：科室讨论、故障原因定性、可靠性级别、WG日期/决议/跟踪、措施完成情况、完成状态
- **关键字信息**：ECAM警告、故障信息、故障源

### 8. 数据导出
将当前筛选结果导出为Excel文件，文件名格式：`工程调查看板_筛选结果_YYYYMMDD_HHmmss.xlsx`。

## 使用方法

### 1. 打开看板
直接在浏览器中打开 `index.html`。支持 Chrome 90+ / Firefox 88+ / Safari 14+ / Edge 90+。

### 2. 上传Excel文件

**方式一：直接上传（推荐）**
拖拽 `.xlsx` 文件到上传区域，或点击选择文件。文件必须包含名为"调查项目清单"的工作表。

**方式二：从HTTP/HTTPS服务器获取**
输入服务器地址（如 `http://192.168.1.100:8080/data.xlsx`），点击"获取"。首次获取后地址保存在本地，可点击"刷新数据"更新。

**方式三：本地文件服务**
```bash
python file-server.py              # 当前目录
python file-server.py ~/Documents  # 指定目录
```
然后在看板中输入 `http://localhost:8765/文件名.xlsx` 获取。

### 3. 使用筛选
展开筛选面板，点击筛选项下拉框多选条件。筛选实时生效，联动所有图表、看板和统计数字。

### 4. 查看详情
点击任意项目卡片 → 查看完整信息 → 点击"返回总览"回到看板。

### 5. 导出数据
设置好筛选条件后，点击"导出筛选结果"按钮下载Excel文件。

## 状态判断逻辑

按优先级从高到低自动判断：
1. 有 `reliability_date` → **可靠性周会**
2. 有 `wg_decision` → **WG周会**
3. 有 `dept_discussion` → **科室讨论**
4. 否则 → **调查中**

## 颜色分区逻辑

```
threshold = ceil(调查天数 × 2/3)

count > 调查天数 → 超期（红）
count > threshold → 注意（橙）
count ≤ threshold → 正常（绿）
```

调查天数或count无效时 → 归类为"已完成数据"。

## 技术栈

- **前端**：纯HTML + JavaScript（无框架，无需构建）
- **UI**：Tailwind CSS (CDN)
- **Excel解析**：SheetJS (xlsx.js CDN)
- **图表**：Chart.js (CDN)
- **部署**：单文件 `index.html`，浏览器直接打开即可

## 数据安全

- 所有数据在浏览器本地处理，不上传至任何服务器
- 不收集用户信息
