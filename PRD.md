## 1. 项目概述

### 1.1 背景
TMC空客组负责跟踪和管理空客机队（A330、A350、A380等）的工程调查项目。目前使用Excel表格（TMC空客组调查项目跟踪单.xlsx）进行数据管理，缺乏直观的可视化看板来实时监控调查进度、识别超期项目和进行多维度筛选分析。

### 1.2 目标
构建一个纯前端的工程调查看板，实现：
- 实时展示工程调查的总览状态（进行中、超期、即将超期、正常）
- 基于时间间隔的四色分区可视化（红/橙/黄/绿）
- 多维度筛选功能（故障类型、机型、注册号、基地、ATA章节、工程师、日期）
- 工程调查详情查看
- 筛选结果导出为Excel

### 1.3 技术选型
- **架构**: 纯前端单页应用（Single HTML File）
- **数据解析**: SheetJS (xlsx.js) - 浏览器端解析Excel文件
- **UI框架**: Tailwind CSS (CDN引入)
- **导出功能**: SheetJS (支持导出筛选结果为Excel)
- **部署**: 无需后端，直接打开HTML文件即可使用

## 3. 核心功能需求

### 3.1 数据上传与解析

**功能描述**: 用户通过拖拽或点击上传Excel文件，系统在前端解析数据。

**交互流程**:
1. 页面顶部显示上传区域（支持拖拽和点击选择）
2. 用户选择或拖入Excel文件
3. 系统使用SheetJS解析`调查项目清单`工作表
4. 解析成功后，自动跳转到总览页面
5. 解析失败时，显示错误提示

**数据校验**:
- 验证文件类型为.xlsx
- 验证工作表名称包含`调查项目清单`
- 验证关键列存在（start_date, fleet, ac等）

### 3.2 状态判断逻辑

**功能描述**: 根据Excel中的特定列自动判断工程调查的当前状态。

**判断规则**（优先级从高到低）:
```
IF 可靠性周会日期 (Y列) 有值:
    status = "可靠性周会"
ELSE IF WG会议决议 (V列) 有值:
    status = "WG周会"
ELSE IF 科室讨论日期/结果 (R列) 有值:
    status = "科室讨论"
ELSE:
    status = "调查中"
```

**注意**: 此逻辑会覆盖Excel中A列的status值，以确保状态判断的准确性。

### 3.3 超期判断逻辑

**功能描述**: 基于工程调查开始日期（start_date）和当前日期计算时间间隔，判断是否超期。

**计算规则**:
```
days_elapsed = (当前日期 - start_date) 的天数差
```

**超期分类**:
| 分类 | 条件 | 颜色标识 | 说明 |
|------|------|----------|------|
| 超期 | days_elapsed > 12 | 红色 | 已超期，需要关注 |
| 即将超期 | 10 ≤ days_elapsed ≤ 12 | 橙色 | 接近超期，需预警 |
| 正常 | days_elapsed ≤ 9 | 根据四色分区 | 正常进行中 |

### 3.4 四色时间分区

**功能描述**: 根据时间间隔将工程调查分为四个颜色区间，直观展示进度。

**分区规则**:
| 颜色 | 时间间隔 | 含义 |
|------|----------|------|
| 🔴 红色 | > 9天 | 时间较长，需重点关注 |
| 🟠 橙色 | 6-8天 | 时间中等，需关注 |
| 🟡 黄色 | 3-5天 | 时间较短 |
| 🟢 绿色 | 0-2天 | 刚开始，时间充裕 |

**注意**: 四色分区独立于超期判断，两者同时展示。

### 3.6 筛选功能

**功能描述**: 支持多维度筛选，快速定位目标工程调查。

**筛选维度**:
| 筛选器 | 类型 | 数据来源 | 说明 |
|--------|------|----------|------|
| 故障类型 (source) | 多选下拉 | C列唯一值 | S/M, G, W, OTHER等 |
| 机型 (fleet) | 多选下拉 | E列唯一值 | A330, A350, A380等 |
| 飞机注册号 (ac) | 多选下拉/搜索 | F列唯一值 | B-30EA, B-320S等 |
| 执管基地 (base) | 多选下拉 | G列唯一值 | 广州, 深圳, 北京等 |
| ATA章节 (ata) | 多选下拉 | I列唯一值 | 21, 24, 27, 29, 32等 |
| TMC工程师 | 多选下拉 | L列唯一值 | 范俊, 安纯阳, 齐红等 |
| 开始日期范围 | 日期范围选择器 | N列 | 选择start_date范围 |

**交互逻辑**:
- 多个筛选条件之间为"与"关系（AND）
- 每个筛选器支持多选（OR关系）
- 提供"重置筛选"按钮，清空所有筛选条件
- 筛选结果实时更新统计卡片和列表

**功能描述**: 将当前筛选结果导出为Excel文件。

**导出内容**:
- 导出当前筛选条件下的所有工程调查记录
- 包含所有关键字段（注册号、机型、基地、ATA、工程师、状态、时间间隔、故障描述等）
- 文件名格式: `工程调查看板_筛选结果_YYYYMMDD_HHmmss.xlsx`

**交互流程**:
1. 用户点击"导出筛选结果"按钮
2. 系统使用SheetJS生成Excel文件
3. 自动触发浏览器下载

## 4. 非功能需求

### 4.1 性能要求
- Excel文件解析时间 < 3秒（580行数据）
- 页面渲染时间 < 1秒
- 筛选响应时间 < 500ms

### 4.2 兼容性
- 支持主流浏览器：Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 支持Windows、macOS、Linux操作系统

### 4.3 用户体验
- 响应式设计，支持桌面端和平板端
- 加载状态提示（解析中、渲染中）
- 错误提示友好（文件格式错误、解析失败等）
- 空状态提示（无数据、筛选无结果等）

### 4.4 数据安全
- 所有数据处理在浏览器本地完成，不上传服务器
- 不收集用户数据或文件内容
